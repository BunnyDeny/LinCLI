# 尾行模式打印支持


<img width="564" height="558" alt="1" src="https://github.com/user-attachments/assets/6e795dfd-a3ad-4e61-ac5b-8c0998387e5a" />



在嵌入式设备上，后台任务（如传感器采集、定时器中断、网络事件）随时可能输出日志，而用户往往正在命令行中输入调试指令。如果日志直接覆盖或穿插在输入行里，命令内容会被破坏、光标位置错乱，用户体验极差。

LinCLI 的**尾行模式打印**机制彻底解决了这个问题。它是 `cli_printk` 的内建行为，**零配置、开箱即用**。

---

## 为什么嵌入式 CLI 特别需要这个功能

桌面操作系统的终端有完善的行编辑和刷新机制，但嵌入式 CLI 通常运行在裸机或 RTOS 上，资源受限，没有现成的终端模拟器可用。在这种环境下，后台日志和前台输入很容易互相干扰：

- **传感器中断**每 100ms 上报一次温湿度，用户刚输了一半的命令就被冲掉
- **Wi-Fi 事件回调**突然打印连接状态，把用户正在输入的 IP 地址切成两半
- **电机控制任务**输出位置反馈，Tab 补全到一半的候选列表被日志撕碎

这些干扰在调试阶段极其频繁，如果不能优雅处理，开发者会反复面对一团乱的终端，效率低下，甚至误操作。

---

## 具体好处

### 1. 后台日志与前台输入互不干扰

无论后台多么繁忙，用户在命令行输入的内容始终安全。日志会**插入到当前行的上方**，而不是覆盖已有输入。

**Before：**
```
lin@linCli> loglev[INFO] temp = 42.5
```
输入的命令 `loglev` 被日志覆盖，后续按键位置错乱。

**After：**
```
[INFO] temp = 42.5
lin@linCli> loglev
```
日志独占一行，命令行完整保留，光标位置正确。

### 2. Tab 补全的候选列表不会被日志破坏

按 `Tab` 触发候选列表后，如果此时后台打印日志，列表会被**自动恢复**。用户不需要重新按 Tab，视觉上始终能看到完整的候选信息。

**Before：**
```
lin@linCli> l<Tab>
led    loglevel
[INFO] voltage low!    <-- 日志直接插在候选列表中间，显示错乱
```

**After：**
```
[INFO] voltage low!
lin@linCli> l
led    loglevel
```

候选列表被完整重绘，用户体验连贯。

### 3. 光标位置始终正确

日志输出后，光标会回到用户输入的**实际位置**，而不是停留在日志末尾。用户可以立刻继续输入、删除、移动光标，无需手动调整。

### 4. 降低调试时的心理负担

开发者不必再担心"现在能不能打印日志"或"先关掉某些调试输出再调命令"。任何时刻、任何代码路径都可以放心调用 `pr_info` / `pr_warn` / `pr_err`，框架会处理好显示冲突。

### 5. 提升现场排障效率

在客户现场或产线调试时，设备通常通过串口连接。后台日志持续输出，工程师同时输入命令查询状态。如果终端内容不断被冲乱，很容易看错数据、输错命令。尾行模式保证了**日志可读、输入可控**，大幅降低了误操作风险。

### 6. 与命令历史、方向键无缝协作

即使日志在打印过程中，用户依然可以正常按 `↑` / `↓` 切换历史记录、按 `←` / `→` 移动光标。历史记录调出后，命令行重绘逻辑同样会正确处理，不会出现残留字符或光标漂移。

---

## 使用方式

**无需任何配置。**

只要通过 `pr_info`、`pr_warn`、`pr_err` 等宏输出日志即可：

```c
pr_info("temperature: %.1f\r\n", temp);
pr_warn("voltage low!\r\n");
pr_err("sensor offline\r\n");
```

> **建议**：日志内容以 `\r\n` 结尾，这样框架可以正确处理光标位置。如果省略换行符，日志和命令提示符会连接在同一行，虽然不会破坏输入内容，但可读性稍差。

---

## 适用场景总结

| 场景 | 尾行模式的价值 |
|---|---|
| 传感器中断频繁上报 | 日志不冲乱输入，调试命令可连续输入 |
| 网络/Wi-Fi 状态回调 | 连接日志不影响正在输入的网络诊断命令 |
| 定时任务周期性输出 | 心跳日志与命令行共存，互不干扰 |
| 多命令链调试 | `&&` 链执行过程中如有后台日志，提示符始终干净 |
| 现场串口调试 | 复杂环境下终端保持可读，降低误操作 |
| Tab 补全复杂命令 | 长命令补全过程中不怕后台日志打断 |

LinCLI 的尾行模式打印让嵌入式 CLI 真正具备了**工业级交互体验**：后台再忙，前台始终清爽可控。
