# cli
致力于实现一个C语言版本的简单，易用，开发者友好型的cli框架

## 开发者视角的命令解析器
命令解析器要完成的任务：

（1）对于命令的每个选项，其都应该有一个参数列表，例如'gcc -o main.o'，这里的-o选项有一个参数main.o，也可以选择不指定任何参数，此时有缺省处理
（2）解析到对应的选项之后，仅仅是赋值某个选项结构体对象的对应字段，该字段本身也是开发者开发期间设计的，但是要遵循合理的规范，例如，规定字段必须要有表示选项名字的成员，以及选项的参数列表（argc，argv），然后每个命令有多个选项，因此需要多个这样的结构体对象，可以考虑传递指针首地址以及对象数量来实现。考虑每个选项都绑定一个默认的预解析函数，用于检查选项参数，以及一些预处理操作，例如提前赋值对应的全局对象的相关信息成员，最终真正的解析函数负责根据这些信息来执行最终的具体操作，这个选项的预解析功能的主要目的是为了检查选项参数以及解析对最终执行环节（由最终的解析函数调用）有用的关键信息，如果某个选项参数不正确则直接返回错误信息，只有所有选项的预解析函数正确执行才会执行最终的解析函数，最终的解析函数主要任务就不是解析具体的选项了，而是根据预解析函数记录的信息来执行最终期望的程序，即本框架将繁杂的字符串解析工作分发给了多个预解析函数，最终的解析函数只负责根据全局结构体记录的信息执行即可，这样大大简化了字符串解析的力

（3）解析函数的实现 当（2）确定好之后，解析函数就可以方便地查看用户输入的各个选项及其对应的参数，为了进一步方便实现解析函数，考虑继续实现一系列方便解析字符串的操作
（4）上述提到的解析函数以及解析选项都是不同命令的个性化配置，所以两者都要交给开发者实现，因此需要提供段注册机制由链接脚本自动收集并编译这些信息，减轻开发者的注册初始化这些步骤的压力，专注于设计业务逻辑的实现细节
（5）用户输入的命令要遵循一定的格式规范才能有效实现（2）目前想到的有：
    - 选项必须以'-'或'--'+英文单词或英文字符的形式给定
